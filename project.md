# Group Project
## Objectives


The objective of this group project is to get a hand-on experience in a team of data scientists, applying what you learn from the class to a real-world data set.  The data set is **housing** , which has a rich set of features and can be downloaded here.  

**Data set** : https://www.kaggle.com/competitions/house-prices-advanced-regression-techniques/data


The project is done in a **group of one or two or three students**. The key tasks are to process the data, build  statistical models predict the house prices and write an evaluation analysis on the goodness of the models.  

You are expected to apply all of the following statistical methods (covered in the class). The notebook is structured in 5 parts that build on each other:

1. **Classical statistical inference.** Sample mean, variance, confidence interval, and hypothesis testing. This can be applied on all the features or housing values themselves.
2. **ANOVA.** Apply ANOVA on the 10 given ordinal features (listed in the notebook) to determine which features have a statistically significant effect on SalePrice. Identify significant main effects and interactions.
3. **2^k Factorial design.** Select factors from the significant features found by ANOVA and apply a factorial design analysis.
4. **Parametric regression.** Build regression models using only the significant ordinal features from ANOVA plus 2 numerical features (`GrLivArea`, `TotalBsmtSF`). Analyze the model outputs and factor contributions.
5. **Non-parametric model (Neural Network).** Fit a regression model based on a neural network using **all** available features. This can be achieved with PyTorch, sklearn MLPRegressor, or similar packages. This model produces the `submission.csv` used for Kaggle scoring.

### Prediction correctness (20%)

The `submission.csv` must be generated by your **neural network model** (Part 5), which uses all features. Your final project repository must include this prediction file with your model's predictions for the test set (`test.csv`). The file must contain two columns: `Id` and `SalePrice`. Your notebook code should produce this file as its output.

We will submit your `submission.csv` to the Kaggle competition and grade based on the RMSE score.

Kaggle evaluates submissions using RMSE on log-transformed prices: $\text{RMSE} = \sqrt{\frac{1}{n}\sum_{i=1}^{n}(\log(y_i + 1) - \log(\hat{y}_i + 1))^2}$.

The grading scale for this component:
| Kaggle RMSE (log) | Grade |
|---|---|
| < 0.13 | 100% |
| 0.13 - 0.15 | 90% |
| 0.15 - 0.18 | 75% |
| 0.18 - 0.22 | 60% |
| > 0.22 | 40% |
